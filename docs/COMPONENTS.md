## Storage Components

### PostgreSQLRepository
Primary storage backend for scan history and trends.

**Key Methods:**
- `save_scan_result()`: Store scan results
- `get_scan_history()`: Retrieve historical scans
- `get_trend_analysis()`: Calculate trends over time
- `get_all_tables_summary()`: Dashboard summary data

### CosmosDBRepository (Optional)
Alternative storage backend for cloud-native deployments.

**Configuration:** Set `USE_COSMOSDB=true` in `.env`

## Data Flow

1. **Request** вҶ’ API receives scan request
2. **Load** вҶ’ Scanner loads CSV into DuckDB
3. **Scan** вҶ’ Soda Core executes data quality checks
4. **Profile** вҶ’ Profiler generates statistics
5. **Detect** вҶ’ Anomaly detector analyzes data
6. **Store** вҶ’ Results saved to PostgreSQL
7. **Alert** вҶ’ Notifications sent if failures detected
8. **Report** вҶ’ HTML report generated
9. **Response** вҶ’ API returns scan results
